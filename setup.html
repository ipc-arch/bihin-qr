備品管理アプリ セットアップ手順書

コピペで進められる、Google Sheets + AppSheet プロトタイプ構築ガイド

この手順書は、職場の管理者アカウントで一から備品管理アプリを作るための完全ガイドです。
所要時間は初回で30〜60分、慣れれば20分程度。特別なプログラミング知識は不要です。

📋 作業の全体像

  1. Google Sheetsで備品台帳を作る（10分）
  2. AppSheetでアプリを自動生成（5分）
  3. アプリの見た目・動きを調整（15分）
  4. 確認アクション（棚卸し機能）を追加（10分）
  5. QRコード生成とラベル印刷（10分）
  6. アイコン・ブランディング（5分）
  7. 権限設定と共有（5分）

1Google Sheetsで備品台帳を作る

1-1. 新規スプレッドシート作成

  1. ブラウザで sheets.new を開く
  2. 左上のタイトルを「備品台帳」に変更
  3. シート名（下部タブ）を「Items」に変更（英数字推奨、AppSheet連携が安定）

1-2. ヘッダー行の作成

以下をコピーして、A1セルに貼り付けてください。自動で14列に展開されます。

ID	管理番号	品名	型番	設置場所	管理課	購入日	購入金額	写真	状態	最終棚卸日	棚卸者	QRコード	備考
💡 Tabで区切られたテキストをセルに貼り付けると、自動で各列に分かれて入ります。これがコピペ運用の基本パターン。

1-3. サンプルデータの投入

テスト用に、A2セルをクリックして以下を貼り付けてください（3件分）。

A-0001	A-0001	複合機 MX-3650	SHARP MX-3650FN	事務室	管財課	2022-04-15	580000		使用中			A-0001	
A-0002	A-0002	プロジェクター	EPSON EB-1795F	第2会議室	総務課	2020-09-10	95000		使用中			A-0002	
A-0003	A-0003	ノートPC	ThinkPad X1 Carbon	情報システム課	システム課	2023-03-01	210000		遊休			A-0003	旧職員使用機、遊休化
⚠️ 「管理番号」と「ID」と「QRコード」は同じ値にしておくのが最小構成のコツです。後で複雑化したくなったら分離できます。

1-4. 列の書式設定

列書式手順
G列（購入日）日付列を選択 → 表示形式 → 数字 → 日付
H列（購入金額）通貨（¥）列を選択 → 表示形式 → 数字 → 通貨
K列（最終棚卸日）日時列を選択 → 表示形式 → 数字 → 日時

2AppSheetでアプリを自動生成

2-1. AppSheetにログイン

  1. appsheet.com にアクセス
  2. 右上「Sign in」→ 職場の管理者アカウント（Google Workspace）でログイン
  3. 初回は利用規約に同意

2-2. スプレッドシートからアプリを作成

  1. ダッシュボード右上 「+ Create」 → 「App」 → 「Start with existing data」
  2. アプリ名を 「備品管理」 と入力
  3. Category は 「Office」 を選択
  4. 「Choose your data」 → Google Sheetsを選択 → 先ほど作った「備品台帳」を選択
  5. 1〜2分待つとAppSheetが自動でアプリの骨格を生成
💡 自動生成直後のアプリでも、リスト表示と詳細画面はすでに動く状態です。ここから調整していきます。

3-2. ビュー（画面）の設定

左メニュー UX → Views で画面を整えます。

メインビュー（一覧画面）

  1. 既存の 「Items」 ビューを選択
  2. View type を 「deck」 に変更（写真+タイトルのカード表示）
  3. Primary header: 品名
  4. Secondary header: 設置場所
  5. Summary column: 状態
  6. Main image: 写真

遊休備品ビュー（追加）

  1. 「+ New View」 → Name: 遊休備品
  2. For this data: Items
  3. View type: deck
  4. Position: menu
  5. Row filter condition: [状態] = "遊休"
  6. Icon: 🎁（リサイクル/ギフト系）
💡 これで「遊休備品だけ表示するページ」が自動で追加され、他課の職員が"お下がり候補"を見られます。

4確認アクション（棚卸し機能）を追加

4-1. 「確認」ボタンのアクション作成

  1. 左メニュー Behavior → Actions
  2. 「+ New Action」
  3. Action name: ✓ この備品を確認した
  4. For a record of this table: Items
  5. Do this: Data: set the values of some columns in this row

Set these columns:

以下を2行追加してください。

ColumnValue（式）
最終棚卸日NOW()
棚卸者USEREMAIL()

Appearance設定

項目値
Display name✓ この備品を確認した
Action iconcheck（チェックマーク）
ProminenceDisplay prominently（目立つ位置）

4-2. 確認後の自動フィードバック（オプション）

  1. 同じActionの Needs confirmation? を ON にすると、押す前に確認ダイアログが出せます
  2. Confirmation message: 「&CONCATENATE([品名], "」を確認済みにします。よろしいですか？")

4-3. ヌケモレ検出ビュー

最終棚卸日が古いままの備品を一覧化するビューを作ります。

  1. UX → Views → + New View
  2. Name: 未確認の備品
  3. View type: deck
  4. Row filter: OR(ISBLANK([最終棚卸日]), [最終棚卸日] < TODAY() - 365)
  5. Icon: ⚠️
⚠️ 条件の 365 は「1年以上前に確認されたもの」の意味。半年なら180、四半期なら90に変更可。

5QRコード生成とラベル印刷

5-1. スプレッドシートにQR画像列を追加

Google Sheetsに戻って、新しい列を右端に追加し、1行目にQR画像と入力。2行目（A-0001の行）に以下を貼り付け。

=IMAGE("https://api.qrserver.com/v1/create-qr-code/?size=200x200&data="&ENCODEURL(A2))

入力後、セル右下の青い四角をドラッグして下にコピー → 全行にQRコードが自動表示されます。

💡 A列（ID = 管理番号）を埋め込んでいるので、QRを読むと管理番号が直接出ます。AppSheet側のスキャン機能で即検索可能。

5-2. AppSheetでQRスキャン機能を有効化

  1. AppSheetで Data → Columns → 管理番号 の列を選択
  2. 「Scannable?」を ON
  3. 保存

これでアプリ内に「Scan」ボタンが出現し、QRを読むと該当備品がすぐ表示されます。

5-3. 印刷用ラベルシートの作成

既存シール横に貼る用のQRラベルを印刷します。スプレッドシートに 新しいシート を追加（下部の + ボタン）し、名前を ラベル印刷 に変更。

A1セルに以下の式を貼り付けてください。Items シートから自動で引っ張って表示します。

=QUERY(Items!A:C, "SELECT A, C WHERE A IS NOT NULL", 1)

次にC列（QR画像）に、2行目から以下を貼り付け、下にドラッグ：

=IMAGE("https://api.qrserver.com/v1/create-qr-code/?size=150x150&data="&ENCODEURL(A2))

行の高さを150pxくらいに広げ、Ctrl+P で印刷 → 用紙サイズを調整して印刷すれば、QR付き管理番号ラベルの完成です。

💡 よりキレイに印刷したい場合はA-oneラベルシール（62面/84面など）用のレイアウトテンプレートに貼り直すと実用的です。

6アイコン・ブランディング

6-1. アプリアイコンをフクロウに変更

  1. このページから owl-icon-512.png をダウンロード
  2. AppSheetで左メニュー UX → Brand
  3. App logo の欄に画像をアップロード
  4. Primary Color を #2b6cb0（紹介ページと同じブルー）に設定
  5. 保存

6-2. ホーム画面の調整

  1. UX → Options
  2. Starting view: Items（一覧画面を最初に表示）
  3. Show bottom bar: ON

7権限設定と共有

7-1. アプリの共有設定

  1. 左メニュー Users → Users
  2. Add users に職員のメールアドレスを入力（カンマ区切りで一括可）
  3. Role: User（通常職員）または Admin（管財課）
  4. Send invite

7-2. 機密備品の権限分離（必要な場合）

特定の備品を管財課のみ閲覧可にしたい場合：

  1. Security → Security Filters
  2. Filter条件に OR([管理課] = "管財課", USEREMAIL() IN Admins[Email]) のような式を設定
⚠️ セキュリティフィルタの設定は慎重に。最初は権限分離なしで運用し、必要性が出てから追加する方が事故が少ないです。

7-3. 配布

  1. 左メニュー Share
  2. Install link をコピー
  3. 管財課の職員にメール or チャットで送付
  4. 職員は各iPadのブラウザでリンクを開き、ホーム画面に追加

🔧 トラブルシュート

Q. 列の型を変更しても反映されない

右上の「Save」ボタンを押し忘れている可能性。AppSheetは明示的な保存が必要です。

Q. スプレッドシートを編集したのにアプリに反映されない

AppSheetダッシュボードで「Regenerate Structure」を実行（Data → テーブル名横の︙メニュー）。

Q. QR画像がスプレッドシートに表示されない

ENCODEURL() 関数を使っているか確認。特殊文字を含むIDはエンコードが必要です。

Q. アプリが重い

サンプル期間は問題ないが、1000件を超えたら Security Filter で課ごとに絞り込むと速くなります。

Q. 無料プランの制限は?

AppSheet Core（Google Workspace Education Plusに含まれる）は10ユーザーまで無料相当。それ以上はライセンス要確認。
